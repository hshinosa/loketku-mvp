# 1. Lean Canvas: Loketku (Micro-Ticketing)

| Komponen | Penjelasan untuk Loketku |
| :--- | :--- |
| **1. Problem (Masalah)** | 1. Uang tiket tertahan di platform besar sampai event selesai, padahal panitia butuh uang tunai untuk DP vendor.<br>2. Pusing merekap target jualan tiket masing-masing anggota panitia secara manual.<br>3. Pengecekan bukti transfer via WhatsApp/mutasi rekening sangat membuang waktu dan rawan penipuan. |
| **2. Customer Segments (Target Pelanggan)** | 1. Mahasiswa (Panitia BEM, Himpunan, UKM).<br>2. Komunitas hobi (turnamen e-sport lokal, gigs musik indie).<br>3. Penyelenggara workshop/webinar skala kecil-menengah. |
| **3. Unique Value Proposition (UVP)** | **"Platform tiket event kampus dengan pencairan dana harian dan pelacakan jualan panitia otomatis. Bebas pusing cek mutasi, uang langsung bisa dipakai buat DP vendor."** |
| **4. Solution (Solusi)** | 1. **Daily Payout System:** Pencairan dana penjualan tiket setiap hari ke rekening panitia.<br>2. **Referral Tracking:** *Link* unik untuk tiap panitia dan *leaderboard* penjualan otomatis.<br>3. **Automated Payment & E-Ticket:** Verifikasi pembayaran otomatis (QRIS) dan pengiriman tiket instan. |
| **5. Channels (Saluran Distribusi)** | 1. *Direct outreach* ke BEM/Himpunan Mahasiswa di kampus-kampus.<br>2. Media sosial (Instagram/TikTok) dengan konten edukasi "cara bikin event anti-ribet".<br>3. *Word of mouth* (panitia yang puas akan merekomendasikan ke organisasi lain). |
| **6. Revenue Streams (Sumber Pendapatan)** | 1. **Platform Fee:** Biaya admin per tiket yang terjual (misal: Rp 2.000 - Rp 5.000 per tiket, dibebankan ke pembeli).<br>2. **Premium Features:** Biaya langganan untuk fitur tambahan (misal: *custom domain*, analitik mendalam, *blast* email ke peserta). |
| **7. Cost Structure (Struktur Biaya)** | 1. Biaya server dan *hosting*.<br>2. Biaya *payment gateway* (potongan per transaksi dan biaya transfer pencairan dana).<br>3. Biaya *marketing* dan operasional tim. |
| **8. Key Metrics (Metrik Utama)** | 1. Jumlah event yang dibuat per bulan.<br>2. Jumlah tiket yang terjual per event.<br>3. *Customer Acquisition Cost* (CAC) - biaya untuk mendapatkan satu penyelenggara event. |
| **9. Unfair Advantage (Keunggulan Kompetitif)** | Fokus *hyper-local* pada komunitas kampus/lokal dengan fitur pencairan dana harian yang jarang dimiliki platform tiket besar (karena risiko *fraud* yang mereka hindari, namun bisa kita mitigasi dengan pendekatan komunitas). |

# 2. Business Model Canvas (BMC): Loketku (Micro-Ticketing)

| Komponen | Penjelasan untuk Loketku |
| :--- | :--- |
| **1. Customer Segments** | Panitia event kampus (BEM, UKM), penyelenggara gigs lokal, pembuat workshop/webinar. |
| **2. Value Propositions** | Pencairan dana harian untuk modal awal event, pelacakan target jualan panitia otomatis, verifikasi pembayaran otomatis, dan e-ticket instan. |
| **3. Channels** | Pendekatan langsung ke kampus, media sosial, *referral program*. |
| **4. Customer Relationships** | *Self-service* (pengguna bisa melakukan semuanya sendiri), *Customer Support* yang responsif via WhatsApp/Chatbot untuk kendala teknis. |
| **5. Revenue Streams** | Biaya admin per transaksi tiket (dibebankan ke pembeli) dan fitur premium berbayar untuk penyelenggara. |
| **6. Key Resources** | Platform web/aplikasi Loketku, server yang andal, integrasi *payment gateway* (dengan kapabilitas *routing/split payment*), tim *developer* & *marketing*. |
| **7. Key Activities** | Pemeliharaan dan pengembangan platform, *marketing* & *sales* (akuisisi penyelenggara event), *customer support*, manajemen risiko pencairan dana. |
| **8. Key Partnerships** | Penyedia *Payment Gateway* (Midtrans, Xendit, dll), Universitas/Kampus (sebagai mitra strategis), Vendor *scan barcode* (opsional). |
| **9. Cost Structure** | Biaya infrastruktur IT (server), gaji pegawai, biaya *marketing*, biaya transaksi *payment gateway*. |
